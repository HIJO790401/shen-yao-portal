著作權 SCBKR 責任邊界引擎 v1
Copyright SCBKR Responsibility-Boundary Engine v1

這不是單純授權平台，而是授權裁決引擎。它不是只問「能不能用」，而是問：誰在申請、為什麼使用、在哪個邊界內成立、何時失效、失效後誰負責。

This is not a simple licensing platform, but a licensing decision engine. It does not only ask whether content can be used, but who is applying, why it is used, under which boundary it is valid, when it fails, and who bears responsibility afterward.

作品定位 / Project positioning

《著作權 SCBKR 責任邊界引擎 v1》是一個 GitHub Pages 可直接運作的雙語前端 demo，用來展示某一批內容在不同用途邊界下，如何得到不同的授權責任判決。

Copyright SCBKR Responsibility-Boundary Engine v1 is a GitHub Pages-ready bilingual front-end demo showing how one body of content can receive different licensing responsibility judgments under different usage boundaries.

長簡介 / Long summary

系統透過 SCBKR 五維結構（主體、因果、邊界、成本簇、責任），判定一次內容使用事件在何種條件下成立、何時失效，以及失效後責任如何重落。它同時示範 Semantic Responsibility Hash 的生成方式，將授權事件壓成可追蹤的公式型責任簽章。

Through the SCBKR five-dimensional structure — Subject, Cause, Boundary, Cost Cluster, and Responsibility — the system determines under which conditions a usage event is valid, when it fails, and how responsibility is reassigned after failure. It also demonstrates the generation of a Semantic Responsibility Hash, turning a licensing event into a traceable formula-based responsibility signature.
