About / 關於我

把語義治理放在模型輸出之前，而不是出事之後才補救

我目前的工作核心，是把 Semantic Firewall 做成一個真正可操作的治理層：不是單看內容危不危險，而是先判斷這段訊息、這個模型輸出、這條行動建議，是否具備進入人類決策鏈的資格。

這個方向結合 SRCP 與 SCBKR 兩個結構框架，將 Responsibility Mapping、Semantic Audit、Cross-model consistency 與 Decision-chain validation 串成一套可以檢查主體、依據、邊界、因果與責任的審計邏輯。

我關心的不是讓模型「看起來比較安全」，而是讓系統在真正影響人之前，先完成可驗證、可追溯、可問責的結構檢查。

Semantic Firewall

在訊息進入人類判斷之前，先做結構資格審查，避免語意污染、假中立敘事與責任空洞的輸出直接影響決策。

AI Governance Layer

把模型回應從單純生成結果，提升為可檢查的治理流程，讓輸出不只可讀，也能被審計、被追問、被驗證。

SRCP · SCBKR

用結構化框架處理主體、因果、邊界、知識風險與責任鏈，作為跨模型一致性檢驗與決策鏈驗證的基礎。

Core Thesis / 核心主張

先問資格，再談風險

很多系統會把治理理解成「把高風險內容分類出來」，但我更在意的是：一段訊息能不能被允許進入人的判斷流程。這會改變我們看待 AI 安全、詐騙防禦與責任鏈設計的方式。

不是看像不像詐騙，而是看它有沒有資格進入人的決策鏈。
Not only whether a message looks fraudulent, but whether it is qualified to enter a human decision chain.

我的工作不只是在分類風險，而是在問：一段訊息、一個模型輸出、或一條決策路徑，是否具備可驗證的主體、因果、邊界、依據與責任。
My work is not only about classifying risk. It asks whether a message, model output, or decision path contains verifiable subject, cause, boundary, basis, and responsibility.
